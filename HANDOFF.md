> 最後更新：2026-06-03　|　分支：`dev`（主分支 `master`）
> 本筆記專指「租約產生器」這個新工具的進度；學區快查、房貸試算為既有功能，不在此範圍。

---

## 1. 目標（本 session 要完成什麼）

打造個人網站的**第三個工具：租約產生器**，與既有的「林口學區快查」「房貸試算」並列於同一站。

核心需求：
- 讓房仲本人與客戶能**自行填欄位產生住宅租賃契約**，不必每次改整份 Word。
- 產出可**列印成 PDF**、也可**匯出 Word**，版面比照現行紙本租約。
- **附屬設備**可勾選並填數量。
- （第二階段）**上傳辨識**：①身分證正反面自動帶入姓名／身分證字號／戶籍地址；②電傳／謄本自動帶入物件基本資料。

本 session 範圍：**完成第一階段「手動版產生器」**；OCR 屬第二階段，尚未動工。

## 4. 關鍵決策與限制

**已確定的技術選擇**
- 維持專案原則：**純前端、零建置、無框架**（HTML + CSS + 原生 JS）。
- **資料與邏輯分離**：設備/特約/條款/選項等放 `rent-data.js`，行為邏輯放 `rent.html`。
- **沿用全站設計系統**：`../style.css` 的 CSS 變數、`.panel`、`.btn`、`.site-nav`、`.hero`。
- **分階段**：先手動版，OCR 第二階段再做（使用者 2026-06-03 拍板）。
- PDF＝瀏覽器列印；Word＝HTML→`.doc`（皆零依賴，不引入打包工具）。
- 契約範本依據：**內政部 113.7.8 台內地字第11302639334號函修正「住宅租賃契約書」**。

**不要改動 / 要小心的部分**
- `rent/reference/` 內是**真實個資**（姓名、身分證字號 O開頭、電話、銀行帳號），**已 gitignore，切勿提交或在範例中亂填**。
- 範例資料（`loadSample()`）一律用假資料，不要放真實個資。
- 條款文字屬**法定範本**，非必要勿任意刪改，以免影響契約效力。
- 既有的學區（`index.html`/`linkou-data.js`）、房貸（`mortgage.html`/`mortgage-data.js`）邏輯**不要動**。

**OCR 方向（尚未拍板，傾向）**
- 傾向**純前端 Tesseract.js**（影像不外傳、維持零後端、合個資考量），辨識結果僅當「草稿」仍須人工核對。
- 雲端 OCR 會需 API 金鑰＋後端代理、有費用、個資上傳第三方 → 與「零建置」衝突，列為備案。

**踩過的雷**
- docx 文字抽取：正則 `<w:t[^>]*>` 會誤吃 `<w:tab>` 自閉合標籤；要用 `<w:t(?: xml:space="[^"]*")?>` 排除。
- 本機**無可用的 Node / Python**（`python.exe` 是 Windows 商店空殼，實際無法執行）→ 無法跑 JS 工具鏈，**只能用瀏覽器 `file://` 測試**。
- 試過用 PowerShell `HttpListener` 架本機伺服器，遇到 `localhost` 逾時 / 殭屍 listener 卡埠（http.sys 由 System PID 4 持有）。**已放棄，改用 `file://` 直接開**（本頁無 fetch/XHR，file:// 完全可用）。臨時腳本 `rent/_devserver.ps1` 已刪除。

## 5. 待辦（依優先順序）

1. **使用者實測版面回饋**：載入範例→產生合約，比對紙本租約，回報要調整的欄位／順序／文字。
2. **核對條款完整度**：第一階段為求精簡，部分有微調——第五條費用略去「(六)其他費用」、第四條(二)押金抵充條文略簡、附件一現況確認書收斂為重點列。**正式使用前應逐條與內政部原範本核對**，確認法律文字無遺漏。
3. **附屬設備清單增補**：確認 `EQUIPMENT` 品項齊全（目前涵蓋窗簾、冰箱、洗衣機、電視、沙發、餐桌椅、床架、衣櫃、吊衣櫃、廚房置物櫃、流理台、瓦斯爐、排油煙機、洗碗機、熱水器、濾水器、冷氣、冷氣遙控器、免治馬桶遙控器、監控設備、燈飾、天然瓦斯）。
4. **多名當事人**：目前出租人／承租人各**僅支援一名**；若有共有人或多名承租人，需擴充欄位（待評估）。
5. **導覽列三頁互通**：在 `index.html`、`mortgage.html` 導覽列加「租約產生器」連結——**使用者要求先暫緩**（貸款+租約仍在調整、尚未上架）。
6. **第二階段 OCR**：①身分證正反面 ②電傳/謄本，先與使用者決定辨識方式（傾向 Tesseract.js）再實作；務必保留人工核對。
7. **版面穩定後 git commit** 建立存檔點。

## 6. 環境

- **技術棧**：純前端 HTML + CSS + 原生 JavaScript，**無框架、無建置流程**。直接開檔即可執行。
- **租約工具目前用到的外部資源**：僅 Google Fonts（Noto Sans TC / Noto Serif TC，CDN）。合約本文字型優先 `標楷體 / DFKai-SB`（系統字型，無需下載）。
- **本工具目前無任何後端 / 外部 API 呼叫。**（全站其他頁另有用 Leaflet 1.9.4、OpenStreetMap、Nominatim，與本工具無關。）
- **第二階段 OCR 可能引入**：Tesseract.js（CDN，純前端）為傾向方案；雲端 OCR API 為備案（需金鑰＋後端，列為次選）。
- **平台 / 工具**：Windows 11、PowerShell；本機**無 Node、無可用 Python**。測試方式＝瀏覽器開 `file:///C:/repo/my-project/rent/rent.html`。
- **版本控管**：Git，目前分支 `dev`，主分支 `master`。
